Dostępność w CSS

1. Kontrast kolorów

Zapewnienie odpowiedniego kontrastu między tekstem a tłem jest kluczowe dla czytelności.

Przykład:

/* Niewystarczający kontrast */
.low-contrast {
    color: #777;
    background-color: #fff;
}

/* Lepszy kontrast */
.high-contrast {
    color: #333;
    background-color: #fff;
}
Wskazówka: Używaj narzędzi takich jak WebAIM Color Contrast Checker do weryfikacji kontrastu.

2. Responsywne jednostki tekstu

Używanie responsywnych jednostek pozwala na lepsze skalowanie tekstu na różnych urządzeniach.

Przykład:

body {
    font-size: 16px; /* Bazowa wielkość czcionki */
}

h1 {
    font-size: 2rem; /* 2 * bazowa wielkość czcionki */
}

p {
    font-size: 1rem;
}

@media (max-width: 768px) {
    body {
        font-size: 14px; /* Mniejsza bazowa wielkość na mniejszych ekranach */
    }
}

3. Fokus i nawigacja klawiaturą

Zapewnienie widocznego fokusa jest kluczowe dla użytkowników korzystających z klawiatury.

Przykład:

/* Domyślny styl fokusa przeglądarki */
:focus {
    outline: 2px solid blue;
}

/* Niestandardowy styl fokusa */
.custom-focus:focus {
    outline: none;
    box-shadow: 0 0 0 3px rgba(0, 123, 255, 0.5);
}

4. Skip linki

Skip linki pozwalają użytkownikom klawiatury na szybkie przejście do głównej zawartości strony.

Przykład:

.skip-link {
    position: absolute;
    top: -40px;
    left: 0;
    background: #000;
    color: white;
    padding: 8px;
    z-index: 100;
}

.skip-link:focus {
    top: 0;
}

5. Właściwe użycie semantycznego HTML i CSS

CSS powinien wspierać semantyczną strukturę HTML, a nie ją zastępować.

Przykład:

/* Zamiast tego */
.fake-button {
    display: inline-block;
    padding: 10px;
    background-color: blue;
    color: white;
    cursor: pointer;
}

/* Użyj prawdziwego przycisku i stylizuj go */
button {
    padding: 10px;
    background-color: blue;
    color: white;
    border: none;
    cursor: pointer;
}

6. Ukrywanie treści dostępnie

Czasami trzeba ukryć treść wizualnie, ale zachować ją dla czytników ekranu.

Przykład:

.visually-hidden {
    position: absolute;
    width: 1px;
    height: 1px;
    padding: 0;
    margin: -1px;
    overflow: hidden;
    clip: rect(0, 0, 0, 0);
    white-space: nowrap;
    border: 0;
}

7. Dostępne formularze

CSS może pomóc w tworzeniu bardziej dostępnych formularzy.

Przykład:

label {
    display: block;
    margin-bottom: 5px;
}

input[type="text"],
input[type="email"],
textarea {
    width: 100%;
    padding: 10px;
    border: 2px solid #ccc;
    border-radius: 4px;
}

input[type="text"]:focus,
input[type="email"]:focus,
textarea:focus {
    border-color: #007bff;
    box-shadow: 0 0 0 3px rgba(0, 123, 255, 0.25);
}

8. Responsywny design dla dostępności

Responsywny design pomaga w dostępności na różnych urządzeniach.

Przykład:

@media screen and (max-width: 600px) {
    .menu {
        display: flex;
        flex-direction: column;
    }

    .menu-item {
        margin-bottom: 10px;
    }
}

9. Dostępne animacje i przejścia

Niektórzy użytkownicy mogą być wrażliwi na animacje. CSS pozwala na kontrolowanie animacji dla tych użytkowników.

Przykład:

@media (prefers-reduced-motion: reduce) {
    * {
        animation: none !important;
        transition: none !important;
    }
}

10. Dostępne tabele

CSS może pomóc w tworzeniu bardziej czytelnych i dostępnych tabel.

Przykład:

table {
    border-collapse: collapse;
    width: 100%;
}

th, td {
    border: 1px solid #ddd;
    padding: 8px;
    text-align: left;
}

th {
    background-color: #f2f2f2;
    font-weight: bold;
}

@media screen and (max-width: 600px) {
    table, thead, tbody, th, td, tr {
        display: block;
    }

    thead tr {
        position: absolute;
        top: -9999px;
        left: -9999px;
    }

    tr {
        margin-bottom: 10px;
    }

    td {
        border: none;
        position: relative;
        padding-left: 50%;
    }

    td:before {
        position: absolute;
        top: 6px;
        left: 6px;
        width: 45%;
        padding-right: 10px;
        white-space: nowrap;
        content: attr(data-label);
        font-weight: bold;
    }
}
Wskazówka: Pamiętaj, że dostępność to nie tylko CSS. Ważne jest również prawidłowe użycie semantycznego HTML i ARIA attributes. CSS powinien wspierać i wzmacniać dostępność, a nie być jedynym narzędziem do jej osiągnięcia.
