Docker Registry – Praktyczne wskazówki

1. Co to jest Docker Registry?

Docker Registry to usługa, która umożliwia przechowywanie, zarządzanie i dystrybucję obrazów kontenerów Docker. Użytkownicy mogą wysyłać (push) swoje obrazy do registry i pobierać (pull) je do uruchomienia na swoich maszynach.

2. Docker Hub

Docker Hub to publiczne registry oferowane przez Docker. Umożliwia przechowywanie i dystrybucję obrazów Docker na szeroką skalę. Jest to najpopularniejsze miejsce do hostowania obrazów, zarówno publicznych, jak i prywatnych.

Przykład: Wysyłanie obrazu do Docker Hub


# Logowanie do Docker Hub
docker login

# Tagowanie obrazu przed wysłaniem
docker tag myapp:latest username/myapp:latest

# Wysyłanie obrazu do Docker Hub
docker push username/myapp:latest

W tym przykładzie logujemy się do Docker Hub, tagujemy obraz myapp i wysyłamy go do naszego konta na Docker Hub.

Pobieranie obrazu z Docker Hub:


docker pull username/myapp:latest

3. Lokalny registry

Lokalne registry to instancja Docker Registry uruchomiona na lokalnym serwerze, co umożliwia przechowywanie obrazów w prywatnej infrastrukturze. Jest to szczególnie przydatne w środowiskach, gdzie wymagana jest większa kontrola nad obrazami i prywatnością.

Przykład: Uruchamianie lokalnego registry


# Uruchomienie lokalnego registry
docker run -d -p 5000:5000 --name registry registry:2

# Tagowanie obrazu
docker tag myapp:latest localhost:5000/myapp:latest

# Wysyłanie obrazu do lokalnego registry
docker push localhost:5000/myapp:latest

Ten przykład pokazuje, jak uruchomić lokalne registry na porcie 5000, a następnie wysłać do niego obraz kontenera.

Pobieranie obrazu z lokalnego registry:


docker pull localhost:5000/myapp:latest

4. Przechowywanie i dystrybucja obrazów

Obrazy Docker mogą być przechowywane i dystrybuowane za pomocą zarówno Docker Hub, jak i lokalnych rejestrów. Kluczową funkcjonalnością registry jest możliwość łatwego dostarczania obrazów do różnych środowisk, takich jak deweloperskie, testowe i produkcyjne.

Przykład: Automatyczne pobieranie obrazów podczas uruchamiania kontenera


# Automatyczne pobieranie obrazu z Docker Hub podczas uruchamiania
docker run username/myapp:latest

Gdy uruchamiasz kontener, Docker automatycznie pobierze najnowszy obraz z registry, jeśli nie jest on dostępny lokalnie.

Wskazówka: Używaj lokalnych rejestrów w środowiskach korporacyjnych, aby zachować pełną kontrolę nad obrazami, a Docker Hub jako miejsce do dzielenia się obrazami z większą społecznością.

Podsumowanie

Docker Registry, w tym Docker Hub i lokalne registry, umożliwia przechowywanie i dystrybucję obrazów Docker. Pozwala to na łatwe udostępnianie obrazów aplikacji pomiędzy różnymi środowiskami oraz automatyzację pobierania i aktualizacji obrazów kontenerów.
