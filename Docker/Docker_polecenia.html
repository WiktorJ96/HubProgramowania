Lista poleceń Docker z flagami

docker run

Tworzy i uruchamia nowy kontener na podstawie obrazu. Jeśli obraz nie jest dostępny lokalnie, Docker pobierze go automatycznie.

Przykład: docker run -d -p 80:80 nginx

Często używane flagi:

  • -d, --detach: Uruchamia kontener w tle
  • -p, --publish: Publikuje port kontenera na hoście
  • -v, --volume: Montuje wolumin
  • --name: Przypisuje nazwę kontenerowi
  • -e, --env: Ustawia zmienne środowiskowe
  • --rm: Automatycznie usuwa kontener po zatrzymaniu

docker pull

Pobiera obraz lub repozytorium z rejestru Docker (domyślnie Docker Hub).

Przykład: docker pull ubuntu

Często używane flagi:

  • -a, --all-tags: Pobiera wszystkie oznaczone obrazy w repozytorium
  • --disable-content-trust: Pomija weryfikację obrazu

docker build

Buduje obraz Docker na podstawie pliku Dockerfile.

Przykład: docker build -t moja-aplikacja .

Często używane flagi:

  • -t, --tag: Nadaje nazwę i opcjonalnie tag w formacie 'name:tag'
  • -f, --file: Nazwa pliku Dockerfile (domyślnie './Dockerfile')
  • --no-cache: Nie używa cache podczas budowania obrazu
  • --pull: Zawsze próbuje pobrać nowszą wersję obrazu bazowego

docker ps

Wyświetla listę uruchomionych kontenerów. Użyj flagi -a, aby zobaczyć wszystkie kontenery (również zatrzymane).

Przykład: docker ps -a

Często używane flagi:

  • -a, --all: Pokazuje wszystkie kontenery (domyślnie pokazuje tylko uruchomione)
  • -q, --quiet: Pokazuje tylko numeryczne ID
  • -s, --size: Wyświetla rozmiary plików
  • -eaf: Pokazuje wszystkie kontenery (również zatrzymane) w pełnym formacie, włączając dodatkowe informacje

docker images

Wyświetla listę obrazów Docker dostępnych lokalnie.

Przykład: docker images

Często używane flagi:

  • -a, --all: Pokazuje wszystkie obrazy (domyślnie ukrywa pośrednie obrazy)
  • -q, --quiet: Pokazuje tylko numeryczne ID obrazów
  • --digests: Pokazuje skróty (digests)

docker stop

Zatrzymuje jeden lub więcej uruchomionych kontenerów.

Przykład: docker stop moj-kontener

Często używane flagi:

  • -t, --time: Liczba sekund do oczekiwania przed zatrzymaniem (domyślnie 10)

docker exec

Uruchamia polecenie w działającym kontenerze.

Przykład: docker exec -it moj-kontener bash

Często używane flagi:

  • -i, --interactive: Utrzymuje STDIN otwarty nawet jeśli nie jest podłączony
  • -t, --tty: Przydziela pseudo-TTY
  • -e, --env: Ustawia zmienne środowiskowe
  • -w, --workdir: Ustawia katalog roboczy wewnątrz kontenera

docker logs

Wyświetla logi kontenera.

Przykład: docker logs moj-kontener

Często używane flagi:

  • -f, --follow: Śledzi logi na bieżąco
  • --tail: Liczba linii do pokazania z końca logów
  • -t, --timestamps: Pokazuje znaczniki czasu

docker-compose

Narzędzie do definiowania i uruchamiania aplikacji Docker składających się z wielu kontenerów.

Przykład: docker-compose up -d

Często używane flagi:

  • up: Tworzy i uruchamia kontenery
  • down: Zatrzymuje i usuwa kontenery, sieci, woluminy i obrazy
  • -d, --detach: Uruchamia w tle
  • --build: Buduje obrazy przed uruchomieniem kontenerów
  • -f, --file: Określa alternatywny plik compose
Wskazówka: Aby uzyskać pełną listę flag dla konkretnego polecenia, użyj docker [polecenie] --help, np. docker run --help.

Pamiętaj, że Docker stale się rozwija, więc mogą pojawiać się nowe polecenia, opcje lub flagi. Zawsze warto sprawdzać oficjalną dokumentację Docker, aby być na bieżąco z najnowszymi zmianami i funkcjonalnościami.
